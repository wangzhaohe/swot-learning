同学们，如果把网页比作一个人：

  • HTML 是人的骨架（身体结构）。

  • JavaScript 是人的灵魂和肌肉（动作、逻辑、交互）。

  • 例子：你点击一个按钮，弹出一个窗口，这就是 JavaScript 在工作。

二：变量——数据的“盒子”

1. 变量的本质

变量就是一个带标签的盒子，用来装数据。

  • 盒子里的东西 = 数据。

  • 盒子上的标签 = 变量名。

2. 变量的声明与赋值

旧时代（不推荐）：var 新时代（ES6标准，必须遵守）：let（可变），const（不可变）。

代码演示：

// 场景：存钱罐
// let 声明的变量，以后可以改
let money = 100;
console.log("现在的存款是：", money); // 100

// 我花了20块买奶茶
money = money - 20;
console.log("花了20后，存款是：", money); // 80

// const 声明的变量，一旦赋值，打死不能改（常量）
const MY_NAME = "张三";
console.log("我的名字是：", MY_NAME);

// 错误演示：尝试修改常量（打开注释会报错）
// MY_NAME = "李四"; // 报错：Assignment to constant variable.

教学重点：

  • 以后定义变量，先用 const，如果发现这个值后面需要改动（比如计数器、金额），再改成 let。坚决不用 var。

  • 命名规范：

    • 见名知意：age 比 a 好，userName 比 x 好。

    • 驼峰命名法：第一个单词首字母小写，后面单词首字母大写，如 studentScore。

三：数据类型——数据的“物种”

JavaScript 是弱类型语言，变量没有类型，值才有类型。

1. 基本数据类型

  • Number（数字）：整数、小数。

  • String（字符串）：被引号包裹的内容。

  • Boolean（布尔值）：真/假。

2. typeof 操作符

用来检测数据的类型。

代码演示：

// 1. Number 类型
let price = 99.9;
console.log(typeof price); // 输出: number

// 2. String 类型
let goodsName = "华为手机";
console.log(typeof goodsName); // 输出: string

// 重点陷阱：数字字符串
let fakeNumber = "100";
console.log(typeof fakeNumber); // 输出: string (它是披着数字皮的字符串！)

// 3. Boolean 类型
let isLogin = true; // true 表示真，false 表示假
console.log(typeof isLogin); // 输出: boolean
3. Undefined 与 Null

  • Undefined：变量声明了但没赋值（盒子是空的，连空气都没有）。

  • Null：变量赋值为空（盒子是空的，但是人为清空的）。

四：字符串详解与模板字符串

1. 字符串拼接的痛点

以前我们要把变量放进字符串，很麻烦：

let name = "王五";
let age = 20;

// 以前的方式：用 + 号拼接，容易写错
let info = "我叫" + name + "，今年" + age + "岁。";
console.log(info);
2. 模板字符串（ES6神器）

使用反引号 ` （键盘左上角ESC下面那个键）和 ${}。

代码演示

let name = "赵六";
let skill = "JavaScript";

// 模板字符串：保留格式，变量直接嵌入
let intro = `大家好，我是${name}。
我正在学习${skill}。
这节课真的"太充实了"！`;

console.log(intro);

教学重点：强调反引号的便捷性，以后项目里全都要用这个，告别 + 号拼接字符串。

案例 2：变量值交换的魔术

需求：有两个变量 a = 10, b = 20，如何交换它们的值，让 a 变成 20，b 变成 10？

方法一：空杯子法（传统）

let a = 10;
let b = 20;
let temp; // 临时空杯子

temp = a; // 把 a 的水倒入 temp (temp=10, a空了)
a = b;    // 把 b 的水倒入 a (a=20, b空了)
b = temp; // 把 temp 的水倒入 b (b=10)

console.log(`a 现在是: ${a}, b 现在是: ${b}`);
console.log('haha:');

方法二：数组解构法（更现代）

let a = 10;
let b = 20;
[a, b] = [b, a]; // 一行代码搞定交换

console.log(`现代写法 -> a: ${a}, b: ${b}`);

S1：核心精讲（4课时）

一：运算符基础——不只是加减乘除

1. 算术运算符

  • 基础：+ (加)、- (减)、* (乘)、/ (除)。

  • 重点：% (取模/取余数)。

  • 应用场景：判断奇偶数、判断倍数、获取数字的最后一位。

代码演示：

let num = 10;
console.log(num % 2); // 输出 0 -> 偶数
console.log(9 % 2);   // 输出 1 -> 奇数

// 经典面试题：如何获取一个三位数 123 的个位数？
let number = 123;
console.log(number % 10); // 输出 3
2. 赋值运算符

  • =：把右边的值给左边（不是相等！）。

  • 复合赋值：+=, -=, *=, /=。

代码演示：

let money = 100;
// 我想存50块进去
// money = money + 50; // 传统写法
money += 50; // 简写，推荐！
console.log(money); // 150
3. 自增自减运算符（难点）

  • ++：自增1。

  • 前置（i`）与后置（`i）的区别：

  • 单独使用：效果一样，都是 i = i + 1。

  • 参与运算/输出：

  • 后置：先用后加。（先输出当前的值，再加1）。

  • 前置：先加后用。（先加1，再输出新值）。

代码演示：

let a = 10;
console.log(a++); // 输出 10。因为先用了10，a变成了11
console.log(a);   // 输出 11

let b = 10;

console.log(a === b); // false -> 数字10 不全等于 字符串"10"

// 结论：以后判断相等，无脑使用 ===
console.log(1 === 1); // true

1. 逻辑与 (AND) &&

  • 口诀：一假则假。

  • 只有两边都为真，结果才为真。

  • 应用：比如“必须同时满足有身份证且年满18岁才能办卡”。

2. 逻辑或 (OR) ||

  • 口诀：一真则真。

  • 只要有一边为真，结果就为真。

  • 应用：比如“有优惠券或者会员卡都可以打折”。

3. 逻辑非 !

  • 取反。!true 变 false。

4. 短路运算（进阶重点）

  • && 短路：如果左边为假，右边不再执行（因为结果注定是假）。

  • || 短路：如果左边为真，右边不再执行（因为结果注定是真）。

代码演示：

// true || (右边代码不执行)
true || num++;
console.log(num); // 0

for (初始化变量; 条件表达式; 操作表达式) {

生活化解读： * 初始化变量：起跑线准备（比如：圈数计数器 i = 1）。 * 条件表达式：判断是否跑完（比如：i ⇐ 5 吗？如果是，继续跑）。 * 操作表达式：跑完一圈后的动作（比如：圈数加1，i++）。 * 循环体：跑步的动作本身。

  • break：直接结束整个循环。

四：双重循环——循环里的循环

1. 概念理解

  • 外层循环动一次，内层循环要跑完一圈。

  • 类比：

  • 外层循环 = 时针（走一格）。

  • 内层循环 = 分针（走一圈）。

2. 经典案例：九九乘法表

逻辑： * 外层 i 控制行数 (1到9)。 * 内层 j 控制列数 (每行有几个公式)。 * 规律：第 i 行，有 i 个公式。

代码演示：

for (let i = 1; i <= 9; i++) { // 控制行
    let rowStr = ""; // 拼接一行的字符串
    for (let j = 1; j <= i; j++) { // 控制列
        rowStr += `${j} x ${i} = ${i * j}\t`; // \t 是制表符，对齐用
    }
    console.log(rowStr);
}

教学提示：这个代码有点难，学生第一次听可能晕。重点是画图解释“时针分针”关系，不强求所有人立刻能自己写出来，但必须理解执行过程。

function generateGoods(count) {

        goodsList.push(goods);
    }

    console.log(`成功生成 ${count} 条商品数据！`);
}

/**
 * Storage 工具类 - 支持对象序列化与类型安全
 */
class StorageWrapper {
  constructor(storage) {
    this.storage = storage;
  }

  /**
   * 设置值（自动序列化）
   * @param {string} key
   * @param {any} value
   * @param {number} expire 过期时间（分钟），仅localStorage支持
   */
  set(key, value, expire = null) {
    try {
      const data = {
        value: value,
        timestamp: Date.now(),
        expire: expire ? Date.now() + expire * 60 * 1000 : null
      };
      this.storage.setItem(key, JSON.stringify(data));
      return true;
    } catch (e) {
      console.error('Storage set error:', e);
      // 容量已满，清理过期数据
      if (e.name === 'QuotaExceededError') {
        this.clearExpired();
        return this.set(key, value, expire);
      }
      return false;
    }
  }

  /**
   * 获取值（自动反序列化+过期检查）
   * @param {string} key
   * @param {any} defaultValue
   */
  get(key, defaultValue = null) {
    try {
      const item = this.storage.getItem(key);
      if (!item) return defaultValue;

      const data = JSON.parse(item);

      // 检查过期
      if (data.expire && Date.now() > data.expire) {
        this.storage.removeItem(key);
        return defaultValue;
      }

      return data.value;
    } catch (e) {
      console.error('Storage get error:', e);
      return defaultValue;
    }
  }

  /**
   * 删除
   */
  remove(key) {
    this.storage.removeItem(key);
  }

  /**
   * 清空所有
   */
  clear() {
    this.storage.clear();
  }

  /**
   * 清理过期数据（仅localStorage需要）
   */
  clearExpired() {
    const now = Date.now();
    for (let i = 0; i < this.storage.length; i++) {
      const key = this.storage.key(i);
      try {
        const data = JSON.parse(this.storage.getItem(key));
        if (data.expire && now > data.expire) {
          this.storage.removeItem(key);
        }
      } catch (e) {
        // 非JSON格式，跳过
      }
    }
  }

  /**
   * 获取所有key
   */
  keys() {
    const keys = [];
    for (let i = 0; i < this.storage.length; i++) {
      keys.push(this.storage.key(i));
    }
    return keys;
  }
}

/**
 * 购物车状态管理 - 支持多标签页同步
 */
class ShoppingCart {
  constructor() {
    this.STORAGE_KEY = 'cart_data';
    this.listeners = [];

    // 监听其他标签页的修改（BOM特性）
    window.addEventListener('storage', (e) => {
      if (e.key === this.STORAGE_KEY) {
        console.log('其他标签页修改了购物车:', e.newValue);
        this.notifyListeners();
      }
    });
  }

  /**
   * 获取购物车数据
   */
  getItems() {
    const data = localStorage.getItem(this.STORAGE_KEY);
    return data ? JSON.parse(data) : [];
  }

  /**
   * 添加商品
   */
  addItem(product) {
    const items = this.getItems();
    const existing = items.find(item => item.id === product.id);

    if (existing) {
      existing.quantity += product.quantity || 1;
      existing.updatedAt = Date.now();
    } else {
      items.push({
        ...product,
        quantity: product.quantity || 1,
        addedAt: Date.now()
      });
    }

    this.save(items);
    return items;
  }

  /**
   * 更新数量
   */
  updateQuantity(productId, quantity) {
    const items = this.getItems();
    const item = items.find(i => i.id === productId);

    if (item) {
      if (quantity <= 0) {
        return this.removeItem(productId);
      }
      item.quantity = quantity;
      item.updatedAt = Date.now();
      this.save(items);
    }
    return items;
  }

  /**
   * 删除商品
   */
  removeItem(productId) {
    const items = this.getItems().filter(i => i.id !== productId);
    this.save(items);
    return items;
  }

  /**
   * 清空购物车
   */
  clear() {
    localStorage.removeItem(this.STORAGE_KEY);
    this.notifyListeners();
  }

  /**
   * 计算总价
   */
  getTotal() {
    return this.getItems().reduce((sum, item) =>
      sum + (item.price * item.quantity), 0
    );
  }

  /**
   * 保存并通知
   */
  save(items) {
    localStorage.setItem(this.STORAGE_KEY, JSON.stringify(items));
    this.notifyListeners();
  }

  /**
   * 订阅变化（用于UI更新）
   */
  subscribe(callback) {
    this.listeners.push(callback);
    // 立即执行一次，获取初始值
    callback(this.getItems());

    // 返回取消订阅函数
    return () => {
      this.listeners = this.listeners.filter(l => l !== callback);
    };
  }

  notifyListeners() {
    const items = this.getItems();
    this.listeners.forEach(cb => cb(items));
  }
}

/**
 * 表单草稿自动保存（页面刷新/返回恢复）

class FormAutoSave {
  constructor(formId, options = {}) {
    this.formId = formId;
    this.key = `form_draft_${formId}`;
    this.interval = options.interval || 3000; // 3秒自动保存
    this.maxAge = options.maxAge || 30; // 30分钟过期

    this.form = document.getElementById(formId);
    this.timer = null;

    this.init();
  }

  init() {
    // 页面加载时恢复数据
    this.restore();

    // 监听输入变化
    this.form.addEventListener('input', () => this.debounceSave());

    // 页面可见性变化时保存（切回标签页）
    document.addEventListener('visibilitychange', () => {
      if (document.visibilityState === 'hidden') {
        this.save();
      }
    });

    // 提交成功后清除草稿
    this.form.addEventListener('submit', () => {
      this.clear();
    });
  }

  /**
   * 防抖保存
   */
  debounceSave() {
    clearTimeout(this.timer);
    this.timer = setTimeout(() => this.save(), this.interval);
  }

  /**
   * 保存表单数据
   */
  save() {
    const formData = new FormData(this.form);
    const data = {
      values: Object.fromEntries(formData),
      timestamp: Date.now(),
      url: window.location.href
    };

    sessionStorage.setItem(this.key, JSON.stringify(data));
    console.log('表单草稿已保存');
  }

  /**
   * 恢复表单数据
   */
  restore() {
    const saved = sessionStorage.getItem(this.key);
    if (!saved) return;

    try {
      const data = JSON.parse(saved);

      // 检查过期时间
      const age = (Date.now() - data.timestamp) / 1000 / 60; // 分钟
      if (age > this.maxAge) {
        console.log('草稿已过期');
        this.clear();
        return;
      }

      // 检查URL是否匹配（防止不同页面数据混淆）
      if (data.url !== window.location.href) {
        console.log('URL不匹配，跳过恢复');
        return;
      }

      // 填充表单
      Object.entries(data.values).forEach(([name, value]) => {
        const field = this.form.elements[name];
        if (field) {
          if (field.type === 'checkbox') {
            field.checked = value === 'on' || value === true;
          } else if (field.type === 'radio') {
            const radio = this.form.querySelector(`[name="${name}"][value="${value}"]`);
            if (radio) radio.checked = true;
          } else {
            field.value = value;
          }
        }
      });

      console.log('表单草稿已恢复');
    } catch (e) {
      console.error('恢复草稿失败:', e);
    }
  }

  /**
   * 清除草稿
   */
  clear() {
    sessionStorage.removeItem(this.key);
  }
}

/**
 * 跨标签页通信通道（BroadcastChannel的降级方案）
 */
class StorageChannel {
  constructor(channelName) {
    this.channel = channelName;
    this.listeners = new Map();

    // 监听storage事件（BOM特性）
    window.addEventListener('storage', (e) => {
      if (e.key === this.channel && e.newValue) {
        try {
          const message = JSON.parse(e.newValue);
          this.handleMessage(message);
        } catch (err) {
          console.error('消息解析失败:', err);
        }
      }
    });
  }

  /**
   * 发送消息到其他标签页
   */
  postMessage(type, payload) {
    const message = {
      type,
      payload,
      timestamp: Date.now(),
      sender: Math.random().toString(36).substr(2, 9) // 简单ID
    };

    // 使用localStorage触发storage事件
    // 立即删除，避免污染存储空间
    localStorage.setItem(this.channel, JSON.stringify(message));
    setTimeout(() => localStorage.removeItem(this.channel), 100);
  }

  /**
   * 订阅消息
   */
  on(type, handler) {
    if (!this.listeners.has(type)) {
      this.listeners.set(type, new Set());
    }
    this.listeners.get(type).add(handler);

    // 返回取消订阅
    return () => this.listeners.get(type)?.delete(handler);
  }

  handleMessage(message) {
    const handlers = this.listeners.get(message.type);
    if (handlers) {
      handlers.forEach(h => h(message.payload, message));
    }
  }
}

一：网络通信基础

1. 客户端与服务器

  • 生活类比：

  • 客户端（浏览器）：顾客（点菜的人）。

  • 服务器：后厨（做菜的人）。

  • HTTP协议：服务员和后厨沟通的“菜单格式”，页面向后端数据库发送接收数据。

  • 请求与响应：

  • Request（请求）：浏览器发给服务器的内容（我要什么）。

  • Response（响应）：服务器回给浏览器的内容（给你什么）。

2. HTTP 状态码

不需要死记硬背，但要知道常见的： * 200：成功。 * 404：找不到资源（路径错了）。 * 500：服务器内部错误（后厨着火了）。

二：同步与异步

1. 同步

  • 代码排队执行，上一行没做完，下一行等着。

  • 缺点：如果有一行代码要执行很久（比如下载大文件），页面会卡死。

2. 异步

  • 代码不排队，先执行后面的，等事情做完了再通知你。

  • 典型场景：定时器、事件、网络请求。

代码演示：

console.log("1. 开始点餐");

// 异步操作：模拟网络请求
setTimeout(function() {
    console.log("3. 饭菜做好了（异步结束）");
}, 2000);

console.log("2. 先去座位等着（代码继续执行）");

// 输出顺序：1 -> 2 -> 3
// 解释：2不会等3执行完才跑，这就是异步。

三：Promise —— 异步操作的“承诺书”

1. 为什么要用 Promise？（概念与背景）

生活案例类比： * 古代（回调函数时代）：你早上去早餐店买包子。你付了钱，老板说“做好了叫你”。但如果有三个环节（做包子→装袋→拿筷子），你就得一直在那等着，老板喊一步你动一步，代码就会变成层层嵌套的“回调地狱”，像倒三角一样，极难维护。

  • 现代：你点了外卖。下单后，APP给你一个“订单号”。你可以先去打游戏、看视频（异步操作）。等外卖到了（状态变为成功），APP弹窗提醒你去拿；如果送丢了（状态变为失败），APP退款。这个“订单号”就是 Promise 对象。

以前我们嵌套异步操作，代码像倒三角，极难维护。

// 回调地狱示例（不要学）
setTimeout(() => {
    console.log("第一步");
    setTimeout(() => {
        console.log("第二步");
        setTimeout(() => {
            console.log("第三步");
        }, 1000);
    }, 1000);
}, 1000);

核心概念： Promise 是一个对象，它代表了一个未来才会知道结果的事件。它有且只有三种状态： 1. Pending（进行中）：比如外卖正在送。 2. Fulfilled / Resolved（已成功）：外卖送到了。 3. Rejected（已失败）：外卖送丢了。 * 重要规则：状态一旦改变（从 Pending 变为 Resolved 或 Rejected），就永久定格，不会再变了。

2. Promise 基本语法与参数详解

语法：new Promise(executor)

  • 参数 executor：这是一个执行器函数，它在 Promise 创建时立即执行。它接收两个参数：

  • resolve：一个函数，调用它表示“成功”，将状态改为 Resolved。

  • reject：一个函数，调用它表示“失败”，将状态改为 Rejected。

代码演示：封装一个模拟读取文件的 Promise

// 1. 创建 Promise 对象
// resolve 和 reject 也是函数，由JS引擎内部提供
const p = new Promise((resolve, reject) => {
    console.log("1. 承诺开始执行..."); // 这行代码是同步执行的！

    // 模拟一个耗时操作（比如网络请求）
    setTimeout(() => {
        let success = true; // 模拟成功或失败

        if (success) {
            // 成功了，调用 resolve，并传递结果数据
            resolve("数据：这是一张美女图片");
        } else {
            // 失败了，调用 reject，并传递错误信息
            reject("错误：网络连接超时");
        }
    }, 1000);
});

console.log("2. 代码继续向下执行...");
3. Promise 原型方法：处理结果

Promise 对象上有三个核心方法来处理结果。

A. .then(onFulfilled, onRejected) * 作用：指定成功和失败的回调。 * 参数： * 第一个参数：成功的回调函数，接收 resolve 传来的数据。 * 第二个参数：失败的回调函数（可选），接收 reject 传来的错误。

B. .catch(onRejected) * 作用：专门用来捕获错误（推荐用法）。相当于 .then(null, onRejected)。

C. .finally(onSettled) * 作用：无论成功还是失败，最终都会执行（比如隐藏加载动画）。

代码演示：链式调用

p.then(result => {
    // 成功时的处理
    console.log("成功回调：", result); // 输出：数据：这是一张美女图片
}).catch(error => {
    // 失败时的处理
    console.log("失败回调：", error);
}).finally(() => {
    // 不管成功失败，都执行
    console.log("承诺结束，加载动画关闭");
});
4. Promise 静态方法：多任务协作

A. Promise.all([]) —— 团队赛 * 规则：所有 Promise 都成功，才返回成功；只要有一个失败，就返回失败。 * 场景：比如游戏资源加载，图片、音频、模型必须全部下载完才能开始。

代码演示：

let p1 = new Promise((resolve) => setTimeout(() => resolve("图片加载完"), 1000));
let p2 = new Promise((resolve) => setTimeout(() => resolve("音频加载完"), 2000));
let p3 = new Promise((resolve) => setTimeout(() => resolve("模型加载完"), 1500));

// 等待所有完成
Promise.all([p1, p2, p3]).then(results => {
    console.log("全部加载成功：", results);
    // 输出：['图片加载完', '音频加载完', '模型加载完'] (顺序固定)
}).catch(error => {
    console.log("有资源加载失败：", error);
});

B. Promise.race([]) —— 个人赛 * 规则：谁先结束（无论成功失败），就返回谁的结果。 * 场景：请求超时机制。比如请求A和定时器B赛跑，如果B先跑完（时间到），就认为A超时。

代码演示：

let request = new Promise(resolve => setTimeout(() => resolve("请求成功"), 2000));
let timeout = new Promise((_, reject) => setTimeout(() => reject("请求超时"), 1000));

Promise.race([request, timeout]).then(res => {
    console.log(res);
}).catch(err => {
    console.log(err); // 1秒后输出：请求超时
});

四：Fetch API —— 现代网络请求标准

1. Fetch 是什么？

  • Fetch 是浏览器原生提供的网络请求接口，用来替代老旧的 XMLHttpRequest (XHR)。

  • 它基于 Promise 设计，不需要回调函数，写法非常优雅。

2. Fetch 基本语法与参数详解

语法：fetch(url, options).then(…​).catch(…​)

参数 1：url (必须) * 请求的地址字符串。

参数 2：options (可选，配置对象) 如果不写，默认是 GET 请求。常用配置项如下： * method：请求方法，如 'GET'、'POST'、'DELETE'。 * headers：请求头对象。POST 请求通常需要设置 'Content-Type': 'application/json'。 * body：请求体。发送数据时使用（GET 请求没有 body）。通常是 JSON 字符串。

3. 示例一：GET 请求（获取数据）

场景：请求一个公开的笑话接口。

代码演示：

// 1. 直接调用 fetch，返回一个 Promise
fetch('https://api.apiopen.top/getJoke?page=1&count=2&type=text')
  .then(response => {
      // 注意：这里的 response 是一个 Response 对象
      // 它包含了状态码、头信息等，但不包含真正的数据

      console.log(response.status); // 200 (HTTP状态码)

      // 2. 我们需要调用 .json() 方法来解析数据体
      // .json() 方法也是一个异步操作，返回一个新的 Promise
      return response.json();
  })
  .then(data => {
      // 3. 这里拿到的才是真正的 JSON 数据
      console.log("最终数据：", data);
      // 可以在这里进行页面渲染
  })
  .catch(error => {
      // 只有网络断开或域名错误才会进这里
      console.log("网络错误：", error);
  });

重点解释： 为什么要写两个 .then？ * 第一个 .then 拿到的是 HTTP 响应报文（包装盒）。 * response.json() 是拆开包装盒（异步过程）。 * 第二个 .then 拿到的是盒子里面的商品。

4. 示例二：POST 请求（提交数据）

场景：模拟用户登录，提交账号密码。

代码演示：

// 准备数据
let userData = {
    username: "admin",
    password: "123456"
};

fetch('https://mock-api.com/login', {
    // 1. 设置方法
    method: 'POST',

    // 2. 设置请求头
    headers: {
        'Content-Type': 'application/json' // 告诉服务器，我发的是JSON格式数据
    },

    // 3. 设置请求体
    // body 只能接收字符串，所以要用 JSON.stringify() 把对象转成字符串
    body: JSON.stringify(userData)
})
.then(response => response.json())
.then(result => {
    if (result.code === 200) {
        console.log("登录成功");
    } else {
        console.log("登录失败：", result.message);
    }
})
.catch(error => console.error("Error:", error));
5. 示例三：Fetch 的错误处理（陷阱！）

重要概念： Fetch 与 jQuery 不同。在 Fetch 中，HTTP 状态码 404 或 500 并不会触发 .catch。只要服务器有响应，Promise 就是 Resolved 状态。.catch 只捕获网络故障（如断网、DNS解析失败）。

正确处理方式：手动检查 response.ok。

代码演示：

fetch('https://api.com/not-found-page') // 一个不存在的地址
  .then(response => {
      // response.ok 是一个布尔值，状态码 200-299 之间为 true
      if (response.ok) {
          return response.json();
      } else {
          // 手动抛出错误，跳转到 catch
          throw new Error(`服务器错误，状态码：${response.status}`);
      }
  })
  .then(data => console.log(data))
  .catch(error => {
      // 这里可以捕获到 404 错误了
      console.log(error.message); // 输出：服务器错误，状态码：404
  });
6. 示例四：结合 Async/Await 的终极写法

这才是现代企业开发的标准写法，代码像同步一样清晰。

代码演示：

async function getData() {
    try {
        // 1. 发送请求，await 等待响应对象
        let response = await fetch('https://api.apiopen.top/getJoke');

        // 2. 手动检查状态
        if (!response.ok) {
            throw new Error("请求失败");
        }

        // 3. 解析数据
        let data = await response.json();

        // 4. 使用数据
        console.log("获取到的笑话：", data.result);

    } catch (error) {
        // 统一处理错误
        console.log("发生错误：", error);
    }
}

getData();

它是 Generator 函数的语法糖，让异步操作更简单。 * 语法：在函数前加 async 关键字。 * 返回值：返回的一定是一个 Promise 对象。

代码演示：

async function hello() {
    return "你好";
    // 虽然返回的是字符串，但实际上函数返回的是 Promise.resolve("你好")
}

hello().then(res => {
    console.log(res); // 你好
});

二：Await 表达式

1. await 是什么？

await 只能用在 async 函数内部。 它会暂停函数的执行，直到 Promise 返回结果。这就是“以同步的方式写异步”。

代码演示：

function getTime() {
    return new Promise(resolve => {
        setTimeout(() => {
            resolve("时间到了");
        }, 2000);
    });
}

async function test() {
    console.log("开始");

    // 这一行会停住，直到 getTime 返回结果
    let result = await getTime();
    console.log(result); // 2秒后打印：时间到了

    console.log("结束");
}

test();

对比：这比写 .then() 看起来清晰太多了！

三：错误处理

1. try…​catch

如果 await 后面的 Promise 失败了，会抛出异常，我们需要用 try…​catch 捕获。

代码演示：

async function loadData() {
    try {
        let res = await fetch('./data.json');
        let data = await res.json();
        console.log("数据：", data);
    } catch (error) {
        console.log("捕获到错误：", error);
    }
}

TS 增强了类的封装性。 * public（默认）：哪里都能用。 * private：私有，只能在类里面用，外面不能访问。 * protected：受保护，只能在类和子类里用。

class Person {

    private age: number; // 私有属性

    getAge(): number {
        return this.age; // 内部可以访问
    }
}

let p = new Person("张三", 18);

console.log(p.getAge()); // OK

===== 3. 常用类名规律

Tailwind 的类名有极强的规律性，基本是 CSS 属性的缩写。

*   **布局**：`flex`, `grid`, `justify-center`, `items-center`.
*   **间距**：`p-{size}` (padding), `m-{size}` (margin). Size 范围通常是 0-96，代表 0.25rem 到 24rem。
    *   `p-4` = `padding: 1rem`
    *   `mx-auto` = `margin-left: auto; margin-right: auto;`
*   **尺寸**：`w-{size}`, `h-{size}`.
    *   `w-full` = `width: 100%`
*   **颜色**：`bg-{color}-{shade}`, `text-{color}-{shade}`.
    *   `bg-red-500` = 中红色背景
    *   `text-white` = 白色文字
*   **边框**：`border`, `rounded-{size}`.
    *   `rounded-lg` = 大圆角

    // 1. 鼠标移入效果

        // 获取当前星星的索引 (0-4)

$(function() {
    // 封装计算总价的函数
    function calculateTotal() {
        let total = 0;
        // 遍历每一个商品行
        $('.item').each(function() {
            // 获取单价和数量
            // 注意：.text() 获取的是文本，需要转数字
            let price = parseFloat($(this).find('.price span').text());
            let num = parseInt($(this).find('.quantity').val());

            // 计算小计
            let subTotal = price * num;

            // 设置小计 (保留两位小数)
            $(this).find('.sub-total span').text(subTotal.toFixed(2));

            // 累加总价
            total += subTotal;
        });

        // 设置总价
        $('#totalPrice').text('￥' + total.toFixed(2));
    }

    // 1. 加号点击事件
    $('#cartList').on('click', '.plus', function() {
        // 获取当前输入框对象
        let $input = $(this).siblings('.quantity');
        let num = parseInt($input.val());
        $input.val(num + 1);

        // 重新计算
        calculateTotal();
    });

    // 2. 减号点击事件
    $('#cartList').on('click', '.minus', function() {
        let $input = $(this).siblings('.quantity');
        let num = parseInt($input.val());

        if (num > 1) {
            $input.val(num - 1);
            calculateTotal();
        } else {
            alert("数量不能少于1");
        }
    });

    // 初始化计算一次
    calculateTotal();
});

===== 说明

**1. 关于 jQuery 选择器的技巧**：
*   在购物车案例中，使用了 `$(this).siblings('.quantity')`，这是非常典型的“相对查找”思维。不要在循环里给每个 input 加 ID，而是通过按钮的位置去找旁边的元素，这在动态生成的列表中尤为重要。

**2. 关于 Tailwind 的响应式**：
*   在手风琴案例中，你可以尝试在小屏幕下改变布局。例如给容器加上 `md:w-1/2 w-full`，这能让学生直观感受响应式开发的便捷。

**3. 关于数据类型**：
*   购物车案例中必须强调 `parseFloat` 和 `parseInt`。JS 从 HTML 取出来的永远是字符串，直接相加会变成字符串拼接（如 "59" + 1 = "591"），这是初学者最容易犯的错。

            $('#search').val('');
            renderTable();
        });

        // ==========================================
        // 7. 弹窗控制 (新增 & 编辑)
        // ==========================================
        function openFormModal(isEdit, item = {}) {
            if (isEdit) {
                $('#modalTitle').text('编辑商品');
                $('#editId').val(item.id);
                $('#inputName').val(item.name);
                $('#inputPrice').val(item.price);
                $('#inputCategory').val(item.category);
            } else {
                $('#modalTitle').text('新增商品');
                $('#editId').val('');
                $form[0].reset();
            }
            $formModal.removeClass('hidden');
        }

        function closeFormModal() {
            $formModal.addClass('hidden');
            $form[0].reset();
            $('#editId').val('');
        }

        $('#addBtn').click(function() {
            openFormModal(false);
        });

        $('#goodsTable').on('click', '.edit-btn', function() {
            let id = $(this).data('id');
            let item = goodsData.find(g => g.id === id);
            if (item) {
                openFormModal(true, item);
            }
        });

        $('#cancelBtn').click(closeFormModal);

        // ==========================================
        // 8. 表单提交
        // ==========================================
        $form.submit(function(e) {
            e.preventDefault();

            let id = $('#editId').val();
            let name = $('#inputName').val().trim();
            let price = parseFloat($('#inputPrice').val());
            let category = $('#inputCategory').val();

            if (!name || isNaN(price)) {
                // 这里也可以用自定义弹窗，为了简单暂用 alert，如果报错同理需替换
                // 暂时简单处理：直接修改文本颜色提示
                if(!name) $('#inputName').css('border-color', 'red');
                return;
            }

            if (id) {
                let index = goodsData.findIndex(item => item.id == id);
                if (index !== -1) {
                    goodsData[index] = {
                        id: Number(id),
                        name: name,
                        price: price,
                        category: category
                    };
                }
            } else {
                let maxId = goodsData.length > 0 ? Math.max(...goodsData.map(item => item.id)) : 0;
                goodsData.push({
                    id: maxId + 1,
                    name: name,
                    price: price,
                    category: category
                });
            }

            closeFormModal();
            $('#search').val('');
            renderTable();
        });

    </script>
