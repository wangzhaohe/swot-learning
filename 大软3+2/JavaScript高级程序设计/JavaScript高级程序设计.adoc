==== 二：变量——数据的“盒子”


===== 1. 变量的本质
变量就是一个**带标签的盒子**，用来装数据。

*   盒子里的东西 = 数据。

*   盒子上的标签 = 变量名。

===== 2. 变量的声明与赋值
**旧时代（不推荐）**：`var`
**新时代（ES6标准，必须遵守）**：`let`（可变），`const`（不可变）。

**代码演示**：

[source,javascript]
----
// 场景：存钱罐
// let 声明的变量，以后可以改
let money = 100; 
console.log("现在的存款是：", money); // 100

// 我花了20块买奶茶
money = money - 20; 
console.log("花了20后，存款是：", money); // 80

// const 声明的变量，一旦赋值，打死不能改（常量）
const MY_NAME = "张三";
console.log("我的名字是：", MY_NAME);

// 错误演示：尝试修改常量（打开注释会报错）
// MY_NAME = "李四"; // 报错：Assignment to constant variable.
----

**教学重点**：

* 以后定义变量，先用 `const`，如果发现这个值后面需要改动（比如计数器、金额），再改成 `let`。坚决不用 `var`。

* **命名规范**：
  ** 见名知意：`age` 比 `a` 好，`userName` 比 `x` 好。
  ** 驼峰命名法：第一个单词首字母小写，后面单词首字母大写，如 `studentScore`。

==== 三：数据类型——数据的“物种”
JavaScript 是弱类型语言，变量没有类型，**值**才有类型。

===== 1. 基本数据类型
* **Number（数字）**：整数、小数。

* **String（字符串）**：被引号包裹的内容。

* **Boolean（布尔值）**：真/假。

===== 2. typeof 操作符
用来检测数据的类型。

**代码演示**：

[source,javascript]
----
// 1. Number 类型
let price = 99.9;
console.log(typeof price); // 输出: number

// 2. String 类型
let goodsName = "华为手机";
console.log(typeof goodsName); // 输出: string

// 重点陷阱：数字字符串
let fakeNumber = "100"; 
console.log(typeof fakeNumber); // 输出: string (它是披着数字皮的字符串！)

// 3. Boolean 类型
let isLogin = true; // true 表示真，false 表示假
console.log(typeof isLogin); // 输出: boolean
----

===== 3. Undefined 与 Null
*   `Undefined`：变量声明了但没赋值（盒子是空的，连空气都没有）。

*   `Null`：变量赋值为空（盒子是空的，但是人为清空的）。

==== 四：字符串详解与模板字符串


===== 1. 字符串拼接的痛点
以前我们要把变量放进字符串，很麻烦：

[source,javascript]
----
let name = "王五";
let age = 20;

// 以前的方式：用 + 号拼接，容易写错
let info = "我叫" + name + "，今年" + age + "岁。";
console.log(info);
----

===== 2. 模板字符串（ES6神器）
使用反引号 `` ` `` （键盘左上角ESC下面那个键）和 `${}`。

**代码演示**

[source,javascript]
----
let name = "赵六";
let skill = "JavaScript";

// 模板字符串：保留格式，变量直接嵌入
let intro = `大家好，我是${name}。
我正在学习${skill}。
这节课真的"太充实了"！`;

console.log(intro);
----

**教学重点**：强调反引号的便捷性，以后项目里全都要用这个，告别 `+` 号拼接字符串。

[source,javascript]
----
let a = 10;

----

=== S1：核心精讲（4课时）


==== 一：运算符基础——不只是加减乘除


===== 1. 算术运算符
*   基础：`+` (加)、`-` (减)、`*` (乘)、`/` (除)。
*   重点：`%` (取模/取余数)。
    *   **应用场景**：判断奇偶数、判断倍数、获取数字的最后一位。

**代码演示**：
```javascript
let num = 10;
console.log(num % 2); // 输出 0 -> 偶数
console.log(9 % 2);   // 输出 1 -> 奇数

// 经典面试题：如何获取一个三位数 123 的个位数？
let number = 123;
console.log(number % 10); // 输出 3
```

===== 2. 赋值运算符
*   `=`：把右边的值给左边（不是相等！）。
*   复合赋值：`+=`, `-=`, `*=`, `/=`。

**代码演示**：
```javascript
let money = 100;
// 我想存50块进去
// money = money + 50; // 传统写法
money += 50; // 简写，推荐！
console.log(money); // 150
```

===== 3. 自增自减运算符（难点）
*   `++`：自增1。
*   前置（`++i`）与后置（`i++`）的区别：
    *   **单独使用**：效果一样，都是 `i = i + 1`。
    *   **参与运算/输出**：
        *   **后置**：先用后加。（先输出当前的值，再加1）。
        *   **前置**：先加后用。（先加1，再输出新值）。

**代码演示**：
```javascript
let a = 10;
console.log(a++); // 输出 10。因为先用了10，a变成了11
console.log(a);   // 输出 11

let b = 10;
console.log(++b); // 输出 11。直接加1变成11，再输出
```
**教学提示**：告诉学生，如果不确定，就分开写，不要为了省事写复杂代码导致Bug。

==== 二：比较运算符——真与假的较量


===== 1. 比较运算符
结果只有两个：`true` (真) 或 `false` (假)。

*   `>` , `<` , `>=` , `<=` ：数字比较。
*   **雷区**：`==` 与 `===` 的区别。
    *   `==` (双等)：只比较**值**，会自动进行类型转换（默认行为，容易出Bug）。
    *   `===` (三等)：比较**值**和**数据类型**（全等，**强烈推荐**）。

**代码演示**：
```javascript
let a = 10;
let b = "10";

// 雷区演示
console.log(a == b);  // true  -> JS把字符串"10"转成了数字10进行比较
console.log(a === b); // false -> 数字10 不全等于 字符串"10"

// 结论：以后判断相等，无脑使用 ===
console.log(1 === 1); // true
console.log(1 === 2); // false
```

---

==== 三：逻辑运算符——组合条件


===== 1. 逻辑与 (AND) `&&`
*   **口诀**：一假则假。
*   只有两边都为真，结果才为真。
*   **应用**：比如“必须同时满足有身份证且年满18岁才能办卡”。

**代码演示**：
```javascript
let hasID = true;
let isAdult = false;

// 办卡条件
let canApply = hasID && isAdult;
console.log(canApply); // false，因为未成年
```

===== 2. 逻辑或 (OR) `||`
*   **口诀**：一真则真。
*   只要有一边为真，结果就为真。
*   **应用**：比如“有优惠券或者会员卡都可以打折”。

===== 3. 逻辑非 `!`
*   取反。`!true` 变 `false`。

===== 4. 短路运算（进阶重点）
*   **`&&` 短路**：如果左边为假，右边不再执行（因为结果注定是假）。
*   **`||` 短路**：如果左边为真，右边不再执行（因为结果注定是真）。

**代码演示**：
```javascript
let num = 0;
// false && (右边代码不执行)
false && num++;
console.log(num); // 0，num++ 根本没运行

// true || (右边代码不执行)
true || num++;
console.log(num); // 0
```

==== 四：流程控制——If 分支结构


===== 1. if 语句
如果条件为真，执行大括号里的代码。

**语法**：
```javascript
if (条件) {
    // 执行代码
}
```

===== 2. if-else 语句
二选一。

**场景**：如果下雨带伞，否则带墨镜。
```javascript
let weather = "下雨";

if (weather === "下雨") {
    console.log("带伞");
} else {
    console.log("带墨镜");
}
```

===== 3. if-else if-else 语句
多选一。

**场景**：成绩评价系统。
```javascript
let score = 75;

if (score >= 90) {
    console.log("优秀");
} else if (score >= 60) {
    console.log("及格");
} else {
    console.log("不及格，补考！");
}
// 输出：及格
```

==== 一：随堂实战案例


===== 案例 1：闰年判断器
**需求**：用户输入一个年份，判断是否是闰年。
**逻辑**：
1.  能被4整除且不能被100整除。
2.  或者能被400整除。
    *(这两个条件是“或”的关系)*

**参考代码**：
```javascript
let year = 2024;

// 逻辑：(能被4整除 且 不能被100整除) 或者 (能被400整除)
// % 是取余，余数为0就是整除
let isLeap = (year % 4 === 0 && year % 100 !== 0) || (year % 400 === 0);

if (isLeap) {
    console.log(`${year} 是闰年`);
} else {
    console.log(`${year} 是平年`);
}
```

===== 案例 2：简易 ATM 取款机
**需求**：模拟ATM操作。余额1000元。输入取款金额。
1.  如果取款金额 > 余额，提示“余额不足”。
2.  如果取款金额 <= 0，提示“输入错误”。
3.  否则，取款成功，扣除余额并显示最新余额。

**参考代码**：
```javascript
let balance = 1000;
let withdrawMoney = 500; // 试着改成 2000 或 -50 测试

if (withdrawMoney <= 0) {
    console.log("取款金额必须大于0");
} else if (withdrawMoney > balance) {
    console.log("余额不足！您的余额为：" + balance);
} else {
    // 取款成功
    balance = balance - withdrawMoney; // 余额扣除
    console.log(`取款成功！您取了 ${withdrawMoney} 元`);
    console.log(`当前余额：${balance} 元`);
}
```

=== S1：核心精讲（4课时）


==== 一：循环初探——为什么要循环？


===== 1. 生活中的循环
*   **场景**：操场跑圈，跑5圈。
*   **如果不写循环**：
    ```javascript
    console.log("跑第1圈");
    console.log("跑第2圈");
    console.log("跑第3圈");
    console.log("跑第4圈");
    console.log("跑第5圈");
    ```
    *问题*：如果要跑100圈或者10000圈呢？复制粘贴会累死人，而且代码极其丑陋。

===== 2. for 循环语法
**语法结构**：
```javascript
for (初始化变量; 条件表达式; 操作表达式) {
    // 循环体（重复执行的代码）
}
```

**生活化解读**：
*   **初始化变量**：起跑线准备（比如：圈数计数器 `i = 1`）。
*   **条件表达式**：判断是否跑完（比如：`i <= 5` 吗？如果是，继续跑）。
*   **操作表达式**：跑完一圈后的动作（比如：圈数加1，`i++`）。
*   **循环体**：跑步的动作本身。

**代码演示**：跑5圈
```javascript
for (let i = 1; i <= 5; i++) {
    console.log(`正在跑第 ${i} 圈`);
}
console.log("跑完了，累死我了！");
```

==== 二：断点调试——看清循环的“心跳”

**重点**：初学者最晕循环，不知道代码执行顺序。必须用断点调试！

**实操演示**：
1.  在代码行号 `for (let i = 1; ...)` 处点击，打上断点。
2.  刷新页面，代码暂停。
3.  点击“下一步”按钮，观察：
    *   **第1次**：`i` 被创建，值为1 -> 判断条件 `1<=5` (真) -> 执行循环体 -> `i` 变成2。
    *   **第2次**：判断条件 `2<=5` (真) -> 执行循环体 -> `i` 变成3。
    *   ...
    *   **最后**：当 `i` 变成6 -> 判断条件 `6<=5` (假) -> **跳出循环**。

**常见死循环（千万别写）**：
```javascript
// 条件永远为真，浏览器会卡死
for (let i = 1; i > 0; i++) {
    console.log("我停不下来啦！");
}
```
*提示*：如果浏览器卡死，赶紧关掉标签页重开。

==== 三：循环实战与累加求和


===== 1. 循环的常见应用
*   **输出数字**：输出1-100。
*   **累加求和**：计算1到100的总和。

**案例：1到100求和**
**思路**：
*   需要一个“存钱罐”变量 `sum`，初始为0。
*   每循环一次，把数字加到存钱罐里。

```javascript
let sum = 0; // 存钱罐，必须放在循环外面！

for (let i = 1; i <= 100; i++) {
    // sum = sum + i;
    sum += i; // 简写
}

console.log(`1加到100的和是：${sum}`); // 5050
```

===== 2. 循环中的 continue 和 break
*   **continue**：跳过本次循环，直接进入下一次。
    *   *场景*：吃苹果，发现第3个烂了，扔掉，继续吃第4个。
*   **break**：直接结束整个循环。
    *   *场景*：吃苹果，吃到第3个发现里面有半条虫子，恶心得剩下的全不吃了。

**代码演示**：
```javascript
// 吃5个苹果
for (let i = 1; i <= 5; i++) {
    if (i === 3) {
        console.log(`第${i}个苹果烂了，扔掉！`);
        continue; // 跳过第3次循环，不执行后面的“吃苹果”
        // 如果是 break，整个循环就结束了，4和5都不吃了
    }
    console.log(`正在吃第${i}个苹果，真香~`);
}
```

==== 四：双重循环——循环里的循环


===== 1. 概念理解
*   外层循环动一次，内层循环要跑完一圈。
*   **类比**：
    *   外层循环 = 时针（走一格）。
    *   内层循环 = 分针（走一圈）。

===== 2. 经典案例：九九乘法表
**逻辑**：
*   外层 `i` 控制行数 (1到9)。
*   内层 `j` 控制列数 (每行有几个公式)。
*   规律：第 `i` 行，有 `i` 个公式。

**代码演示**：
```javascript
for (let i = 1; i <= 9; i++) { // 控制行
    let rowStr = ""; // 拼接一行的字符串
    for (let j = 1; j <= i; j++) { // 控制列
        rowStr += `${j} x ${i} = ${i * j}\t`; // \t 是制表符，对齐用
    }
    console.log(rowStr);
}
```
*教学提示*：这个代码有点难，学生第一次听可能晕。重点是画图解释“时针分针”关系，不强求所有人立刻能自己写出来，但必须理解执行过程。

===== 案例 1：求偶数和
**需求**：计算1到100之间所有偶数的和。
**思路**：在循环内部加一个 `if` 判断。

```javascript

```

===== 案例 2：简易ATM取款机（循环版）
**需求**：之前的ATM只能操作一次，现在要让它一直运行，直到用户选择“退出”。
**关键技术**：`while (true)` 配合 `break`。

```javascript
let balance = 1000;

while (true) { // 死循环，一直运行
    let choice = prompt("请选择操作：\n1. 查询余额\n2. 存款\n3. 取款\n4. 退出");

    if (choice === "1") {
        alert(`您的余额为：${balance}元`);
    } else if (choice === "2") {
        let money = parseFloat(prompt("请输入存款金额："));
        balance += money;
        alert(`存款成功！余额：${balance}`);
    } else if (choice === "3") {
        let money = parseFloat(prompt("请输入取款金额："));
        if (money > balance) {
            alert("余额不足！");
        } else {
            balance -= money;
            alert(`取款成功！余额：${balance}`);
        }
    } else if (choice === "4") {
        alert("感谢使用，再见！");
        break; // 退出死循环
    } else {
        alert("输入错误，请重试！");
    }
}
```
*注意*：`prompt` 返回的是字符串，如果是数字操作建议用 `parseFloat` 转换。

```javascript
/**
 * 函数：生成测试商品数据
 * 参数：count (生成多少个)
 */
function generateGoods(count) {
    // 清空原数组
    goodsList = [];

    // 商品名称模板
    const names = ["高等数学", "大学英语", "计算机网络", "二手自行车", "台灯", "风扇", "U盘"];

    for (let i = 1; i <= count; i++) {
        // 生成一个随机商品对象
        let goods = {
            id: i,
            name: names[Math.floor(Math.random() * names.length)] + " (第" + i + "件)",
            price: Math.floor(Math.random() * 100) + 10, // 10-110之间的随机价格

        };

        // 把商品推入仓库
        goodsList.push(goods);
    }

    console.log(`成功生成 ${count} 条商品数据！`);
}
```

**知识点解释**：
*   `Math.random()`：生成 0 到 1 之间的随机小数。
*   `Math.floor()`：向下取整（去掉小数部分）。
*   `goodsList.push(goods)`：把生成的对象塞进数组。虽然数组还没细讲，但这里作为容器使用，学生能理解“存进去”即可。

=== S1：核心精讲（4课时）


==== 一：函数初探——不仅是代码块


===== 1. 为什么需要函数？
*   **现状**：前两天我们写的代码，执行一遍就结束了。如果想执行同样的逻辑（比如算折扣），必须把代码复制粘贴一遍。
*   **问题**：代码冗余、难以维护（改一个地方，要改十个地方）。
*   **解决**：把功能像“打包快递”一样封起来，贴个标签（函数名），随时调用。

===== 2. 函数的声明与调用
**语法**：
```javascript
function 函数名() {
    // 函数体：要封装的代码
}

函数名(); // 调用：让函数执行
```

**代码演示**：
```javascript
// 1. 声明函数：定义一个“打招呼”的功能
function sayHi() {
    console.log("----------");
    console.log("你好，欢迎光临！");
    console.log("----------");
}

// 2. 调用函数：想调几次调几次
sayHi(); 
sayHi(); 
// 此时控制台会打印两次问候语。
```

==== 二：参数与返回值——函数的“输入”与“输出”

**生活类比：榨汁机**
*   **参数**：放入的水果。不同的水果榨出不同的汁。
*   **函数体**：榨汁的过程。
*   **返回值**：产出的果汁。

===== 1. 形参与实参
*   **形参**：定义函数时的变量（占位符），相当于“投料口”。
*   **实参**：调用函数时传入的具体值，相当于“具体的苹果”。

**代码演示**：
```javascript
// 定义函数：此时 num1, num2 只是形式上的参数
function getSum(num1, num2) {
    console.log(`正在计算 ${num1} + ${num2}`);
    let result = num1 + num2;
    console.log(`结果是：${result}`);
}

// 调用函数：此时 10, 20 是实际的参数
getSum(10, 20); // 输出 30
getSum(100, 50); // 输出 150
```

===== 2. 返回值
函数执行完后，把结果“吐”出来给调用者。使用 `return` 关键字。

**重点**：
1.  `return` 后面的代码不再执行（函数结束）。
2.  如果没有 `return`，函数默认返回 `undefined`。

**代码演示**：
```javascript
function cook(name) {
    console.log("开始做饭...");
    return name + "做好了！"; // 把结果返回出去
    console.log("这句话不会执行"); // return后面的代码不执行
}

// 接收返回值
let food = cook("红烧肉");
console.log(food); // 输出：红烧肉做好了！
console.log(cook("蛋炒饭")); // 也可以直接打印
```

==== 三：作用域——变量的“地盘”


===== 1. 全局变量
*   在函数**外部**定义的变量。
*   **特点**：任何地方都能用，浏览器关闭才销毁。
*   **缺点**：容易造成全局污染（变量名冲突）。

===== 2. 局部变量
*   在函数**内部**定义的变量。
*   **特点**：只能在函数内部使用，函数执行完就销毁。
*   **优点**：安全，省内存。

**代码演示**：
```javascript
let globalVar = "我是全局变量"; // 全局

function myFunction() {
    let localVar = "我是局部变量"; // 局部
    console.log(globalVar); // 可以访问
    console.log(localVar);  // 可以访问
}

myFunction(); // 必须调用才能看到内部逻辑

console.log(globalVar); // 可以访问
// console.log(localVar);  // 报错！localVar is not defined (在外面找不到)
```

===== 3. 作用域链
变量在内部找不到，会向上一级找，直到全局。这就叫作用域链。

```javascript
let num = 10;
function fn() {
    let num = 20;
    function fn2() {
        console.log(num); // 输出 20 (就近原则，找离得最近的上级)
    }
    fn2();
}
fn();
```

===== 1. 更简洁的写法
现代前端开发（Vue/React）中，箭头函数是主流。

**语法**：
```javascript
const 函数名 = (参数) => {
    // 函数体
    return 结果;
}
```

**代码演示**：
```javascript
// 传统写法
function add(a, b) {
    return a + b;
}

// 箭头函数写法
const addArrow = (a, b) => {
    return a + b;
}

// 极简写法（当函数体只有一行return时，可以省略大括号和return）
const addSimple = (a, b) => a + b;

console.log(addSimple(1, 2)); // 3
```


=== S1：核心精讲（4课时）


==== 一：DOM树与节点关系


===== 1. 什么是DOM？
*   **Document Object Model**（文档对象模型）。
*   **大白话解释**：浏览器加载HTML时，会把HTML里的标签、文字、图片都变成一个个的“对象”，这些对象像树杈一样连在一起，这就叫DOM树。
*   **为什么学它？**：JS想要操作网页（改文字、改颜色），必须先在树里找到这个“对象”。

===== 2. 节点关系
把DOM树想象成家谱：
*   **父节点**：直接包裹你的元素。
*   **子节点**：你直接包裹的元素。
*   **兄弟节点**：和你同级、同一个父元素的元素。

**代码演示**：
```html
<div class="father">
    <div class="son">儿子</div>
    <div class="daughter">女儿</div>
</div>

<script>
    let son = document.querySelector('.son');
    // 找爸爸
    console.log(son.parentNode); // <div class="father">...

    // 找兄弟 (nextElementSibling 下一个兄弟)
    console.log(son.nextElementSibling); // <div class="daughter">...
</script>
```

---

==== 二：获取元素——querySelector 全家桶

以前我们用 `document.getElementById('id')`，太长了，而且只能用ID。现在我们学现代方法，**就像写CSS一样**来选元素。

===== 1. 获取单个元素
语法：`document.querySelector('选择器')`。
*   返回：第一个匹配的元素。如果没有，返回 `null`。

**代码演示**：
```javascript
// 选ID
let nav = document.querySelector('#nav'); 
// 选类名
let item = document.querySelector('.item');
// 选标签
let firstLi = document.querySelector('li');
```

===== 2. 获取所有元素
语法：`document.querySelectorAll('选择器')`。
*   返回：一个伪数组。即使只有一个元素，也是数组。要想操作每一个，必须遍历！

**代码演示**：
```javascript
let allLis = document.querySelectorAll('li');
// 这是一个伪数组，不能直接 allLis.style.color = 'red' (报错！)
allLis.forEach(function(li) {
    li.style.color = 'red'; // 遍历后逐个修改
});
```

==== 三：操作元素内容


===== 1. innerText
*   获取或设置标签里面的**纯文本**。
*   特点：不识别HTML标签，读取时会去掉换行和空格。

===== 2. innerHTML
*   获取或设置标签里面的**HTML代码**。
*   特点：识别标签，保留格式。**开发中最常用**。

**代码演示**：
```html
<div id="box">
    <strong>你好</strong>
</div>

<script>
    let box = document.querySelector('#box');

    console.log(box.innerText);  // 输出: 你好 (强标签没了)
    console.log(box.innerHTML);  // 输出: <strong>你好</strong>

    // 修改内容
    box.innerText = '<em>新的内容</em>'; // 页面显示: <em>新的内容</em> (原样输出)
    box.innerHTML = '<em>新的内容</em>'; // 页面显示: 新的内容 (斜体，em生效了)
</script>
```

==== 四：操作元素样式


===== 1. style 属性（行内样式）
*   语法：`元素.style.属性名 = '值'`。
*   **注意**：JS里的属性名要用驼峰命名法。
    *   CSS: `background-color` -> JS: `backgroundColor`
    *   CSS: `font-size` -> JS: `fontSize`

===== 2. className 类名操作
*   语法：`元素.className = '类名'`。
*   **雷区**：这会覆盖掉原来的类名。

===== 3. classList 类名列表（推荐）
*   `元素.classList.add('类名')`：添加类。
*   `元素.classList.remove('类名')`：移除类。
*   `元素.classList.toggle('类名')`：切换类（有则删，无则加）。

**代码演示**：
```javascript
let box = document.querySelector('#box');

// 1. 行内样式修改

box.style.width = '200px';

// 2. 类名修改
box.classList.add('active'); // 添加 active 类
box.classList.toggle('hide'); // 切换 hide 类
```

==== 二：事件基础——互动的桥梁


===== 1. 事件三要素
*   **事件源**：谁被触发了？（比如：按钮）。
*   **事件类型**：怎么触发的？（比如：点击 click、鼠标经过 mouseover）。
*   **事件处理程序**：触发后做什么？（函数）。

===== 2. 执行事件的步骤

3.  编写处理程序。

==== 三：注册事件两种方式


===== 1. 传统方式
`元素.onclick = function() {}`
*   **特点**：简单，但会被覆盖。后面写的 `onclick` 会覆盖前面的。

===== 2. 方法监听方式（推荐）
`元素.addEventListener('事件类型', function() {})`
*   **特点**：不会被覆盖，可以绑定多个事件。

**代码演示**：
```javascript
let btn = document.querySelector('button');

btn.addEventListener('click', function() {
    alert('事件1');
});
btn.addEventListener('click', function() {
    alert('事件2');
});
// 点击按钮，两个alert都会弹出来
```

---

==== 四：删除事件

*   传统方式：`btn.onclick = null;`
*   监听方式：`btn.removeEventListener('click', 函数名)`。
    *   注意：如果要移除，函数必须有名字，不能是匿名函数。

==== 第三、四课时：贯穿项目推进——《易校园》列表渲染

**场景**：这是前端开发最核心的环节：**数据驱动视图**。我们要把内存里的数组数据，变成网页上的HTML结构。

**步骤 1：准备容器**
HTML中只保留一个空壳：
```html
<div class="goods-list" id="goodsBox">
    <!-- 这里原本是空的 -->
</div>
```

**步骤 2：封装渲染函数**
核心思想：遍历数组 -> 拼接字符串 -> innerHTML 填入。

```javascript
// 模拟数据
let goodsData = [
    { id: 1, name: '高等数学', price: 15 },
    { id: 2, name: '二手吉他', price: 200 }
];

let goodsBox = document.querySelector('#goodsBox');

function renderGoods() {
    // 清空容器
    goodsBox.innerHTML = '';

    // 遍历数组
    goodsData.forEach(item => {
        // 创建 div
        let div = document.createElement('div');
        div.className = 'goods-item';

        // 填充内容
        div.innerHTML = `
            <h4>${item.name}</h4>
            <p>价格：￥${item.price}</p>
            <button data-id="${item.id}">删除</button>
        `;

        // 添加到容器
        goodsBox.appendChild(div);
    });
}

// 页面加载先渲染一次
renderGoods();
```

**步骤 3：实现“发布商品”功能**
```javascript
let addBtn = document.querySelector('#addBtn'); // 假设HTML里有一个添加按钮

addBtn.onclick = function() {
    // 1. 获取用户输入(假设有输入框)
    // ...

    // 2. 模拟新增数据
    let newGoods = {
        id: Date.now(), // 时间戳做ID
        name: '新书' + Math.random(),
        price: 50
    };

    // 3. 数据推入数组
    goodsData.push(newGoods);

    // 4. 重新渲染页面
```

示例 1：基础封装（类型安全版）

```javascript
/**
 * Storage 工具类 - 支持对象序列化与类型安全
 */
class StorageWrapper {
  constructor(storage) {
    this.storage = storage;
  }

  /**
   * 设置值（自动序列化）
   * @param {string} key 
   * @param {any} value 
   * @param {number} expire 过期时间（分钟），仅localStorage支持
   */
  set(key, value, expire = null) {
    try {
      const data = {
        value: value,
        timestamp: Date.now(),
        expire: expire ? Date.now() + expire * 60 * 1000 : null
      };
      this.storage.setItem(key, JSON.stringify(data));
      return true;
    } catch (e) {
      console.error('Storage set error:', e);
      // 容量已满，清理过期数据
      if (e.name === 'QuotaExceededError') {
        this.clearExpired();
        return this.set(key, value, expire);
      }
      return false;
    }
  }

  /**
   * 获取值（自动反序列化+过期检查）
   * @param {string} key 
   * @param {any} defaultValue 
   */
  get(key, defaultValue = null) {
    try {
      const item = this.storage.getItem(key);
      if (!item) return defaultValue;

      const data = JSON.parse(item);

      // 检查过期
      if (data.expire && Date.now() > data.expire) {
        this.storage.removeItem(key);
        return defaultValue;
      }

      return data.value;
    } catch (e) {
      console.error('Storage get error:', e);
      return defaultValue;
    }
  }

  /**
   * 删除
   */
  remove(key) {
    this.storage.removeItem(key);
  }

  /**
   * 清空所有
   */
  clear() {
    this.storage.clear();
  }

  /**
   * 清理过期数据（仅localStorage需要）
   */
  clearExpired() {
    const now = Date.now();
    for (let i = 0; i < this.storage.length; i++) {
      const key = this.storage.key(i);
      try {
        const data = JSON.parse(this.storage.getItem(key));
        if (data.expire && now > data.expire) {
          this.storage.removeItem(key);
        }
      } catch (e) {
        // 非JSON格式，跳过
      }
    }
  }

  /**
   * 获取所有key
   */
  keys() {
    const keys = [];
    for (let i = 0; i < this.storage.length; i++) {
      keys.push(this.storage.key(i));
    }
    return keys;
  }
}

// 导出实例
export const local = new StorageWrapper(localStorage);
export const session = new StorageWrapper(sessionStorage);
```

```javascript
/**
 * 购物车状态管理 - 支持多标签页同步
 */
class ShoppingCart {
  constructor() {
    this.STORAGE_KEY = 'cart_data';
    this.listeners = [];

    // 监听其他标签页的修改（BOM特性）
    window.addEventListener('storage', (e) => {
      if (e.key === this.STORAGE_KEY) {
        console.log('其他标签页修改了购物车:', e.newValue);
        this.notifyListeners();
      }
    });
  }

  /**
   * 获取购物车数据
   */
  getItems() {
    const data = localStorage.getItem(this.STORAGE_KEY);
    return data ? JSON.parse(data) : [];
  }

  /**
   * 添加商品
   */
  addItem(product) {
    const items = this.getItems();
    const existing = items.find(item => item.id === product.id);

    if (existing) {
      existing.quantity += product.quantity || 1;
      existing.updatedAt = Date.now();
    } else {
      items.push({
        ...product,
        quantity: product.quantity || 1,
        addedAt: Date.now()
      });
    }

    this.save(items);
    return items;
  }

  /**
   * 更新数量
   */
  updateQuantity(productId, quantity) {
    const items = this.getItems();
    const item = items.find(i => i.id === productId);

    if (item) {
      if (quantity <= 0) {
        return this.removeItem(productId);
      }
      item.quantity = quantity;
      item.updatedAt = Date.now();
      this.save(items);
    }
    return items;
  }

  /**
   * 删除商品
   */
  removeItem(productId) {
    const items = this.getItems().filter(i => i.id !== productId);
    this.save(items);
    return items;
  }

  /**
   * 清空购物车
   */
  clear() {
    localStorage.removeItem(this.STORAGE_KEY);
    this.notifyListeners();
  }

  /**
   * 计算总价
   */
  getTotal() {
    return this.getItems().reduce((sum, item) => 
      sum + (item.price * item.quantity), 0
    );
  }

  /**
   * 保存并通知
   */
  save(items) {
    localStorage.setItem(this.STORAGE_KEY, JSON.stringify(items));
    this.notifyListeners();
  }

  /**
   * 订阅变化（用于UI更新）
   */
  subscribe(callback) {
    this.listeners.push(callback);
    // 立即执行一次，获取初始值
    callback(this.getItems());

    // 返回取消订阅函数
    return () => {
      this.listeners = this.listeners.filter(l => l !== callback);
    };
  }

  notifyListeners() {
    const items = this.getItems();
    this.listeners.forEach(cb => cb(items));
  }
}
```

```javascript
/**
 * 表单草稿自动保存（页面刷新/返回恢复）
 */
class FormAutoSave {
  constructor(formId, options = {}) {
    this.formId = formId;
    this.key = `form_draft_${formId}`;
    this.interval = options.interval || 3000; // 3秒自动保存
    this.maxAge = options.maxAge || 30; // 30分钟过期

    this.form = document.getElementById(formId);
    this.timer = null;

    this.init();
  }

  init() {
    // 页面加载时恢复数据
    this.restore();

    // 监听输入变化
    this.form.addEventListener('input', () => this.debounceSave());

    // 页面可见性变化时保存（切回标签页）
    document.addEventListener('visibilitychange', () => {
      if (document.visibilityState === 'hidden') {
        this.save();
      }
    });

    // 提交成功后清除草稿
    this.form.addEventListener('submit', () => {
      this.clear();
    });
  }

  /**
   * 防抖保存
   */
  debounceSave() {
    clearTimeout(this.timer);
    this.timer = setTimeout(() => this.save(), this.interval);
  }

  /**
   * 保存表单数据
   */
  save() {
    const formData = new FormData(this.form);
    const data = {
      values: Object.fromEntries(formData),
      timestamp: Date.now(),
      url: window.location.href
    };

    sessionStorage.setItem(this.key, JSON.stringify(data));
    console.log('表单草稿已保存');
  }

  /**
   * 恢复表单数据
   */
  restore() {
    const saved = sessionStorage.getItem(this.key);
    if (!saved) return;

    try {
      const data = JSON.parse(saved);

      // 检查过期时间
      const age = (Date.now() - data.timestamp) / 1000 / 60; // 分钟
      if (age > this.maxAge) {
        console.log('草稿已过期');
        this.clear();
        return;
      }

      // 检查URL是否匹配（防止不同页面数据混淆）
      if (data.url !== window.location.href) {
        console.log('URL不匹配，跳过恢复');
        return;
      }

      // 填充表单
      Object.entries(data.values).forEach(([name, value]) => {
        const field = this.form.elements[name];
        if (field) {
          if (field.type === 'checkbox') {
            field.checked = value === 'on' || value === true;
          } else if (field.type === 'radio') {
            const radio = this.form.querySelector(`[name="${name}"][value="${value}"]`);
            if (radio) radio.checked = true;
          } else {
            field.value = value;
          }
        }
      });

      console.log('表单草稿已恢复');
    } catch (e) {
      console.error('恢复草稿失败:', e);
    }
  }

  /**
   * 清除草稿
   */
  clear() {
    sessionStorage.removeItem(this.key);
  }
}

// HTML使用
// <form id="productForm">
//   <input name="title" />
//   <textarea name="description"></textarea>
// </form>

// JS初始化

  interval: 2000,  // 2秒保存一次
  maxAge: 60       // 1小时过期
});
```

```javascript
/**
 * 跨标签页通信通道（BroadcastChannel的降级方案）
 */
class StorageChannel {
  constructor(channelName) {
    this.channel = channelName;
    this.listeners = new Map();

    // 监听storage事件（BOM特性）
    window.addEventListener('storage', (e) => {
      if (e.key === this.channel && e.newValue) {
        try {
          const message = JSON.parse(e.newValue);
          this.handleMessage(message);
        } catch (err) {
          console.error('消息解析失败:', err);
        }
      }
    });
  }

  /**
   * 发送消息到其他标签页
   */
  postMessage(type, payload) {
    const message = {
      type,
      payload,
      timestamp: Date.now(),
      sender: Math.random().toString(36).substr(2, 9) // 简单ID
    };

    // 使用localStorage触发storage事件
    // 立即删除，避免污染存储空间
    localStorage.setItem(this.channel, JSON.stringify(message));
    setTimeout(() => localStorage.removeItem(this.channel), 100);
  }

  /**
   * 订阅消息
   */
  on(type, handler) {
    if (!this.listeners.has(type)) {
      this.listeners.set(type, new Set());
    }
    this.listeners.get(type).add(handler);

    // 返回取消订阅
    return () => this.listeners.get(type)?.delete(handler);
  }

  handleMessage(message) {
    const handlers = this.listeners.get(message.type);
    if (handlers) {
      handlers.forEach(h => h(message.payload, message));
    }
  }
}

// 使用：实现多标签页登录状态同步
const authChannel = new StorageChannel('app_auth');

// 标签页A：监听登出
authChannel.on('logout', () => {

  alert('您已在其他标签页退出登录');
});

// 标签页B：执行登出
document.getElementById('logout').addEventListener('click', () => {
  // 清除登录状态
  localStorage.removeItem('token');
  // 通知其他标签页
```

| 问题             | 原因                      | 解决方案                                     |
| ---------------- | ------------------------- | -------------------------------------------- |
| **存储容量超限** | 超过5-10MB限制            | 封装时捕获`QuotaExceededError`，清理过期数据 |
| **只能存字符串** | API设计限制               | 使用`JSON.stringify/parse`序列化             |
| **同步阻塞**     | Storage操作是同步的       | 大数据量时改用IndexedDB                      |
| **隐私模式失效** | Safari隐私模式禁用Storage | 使用`try-catch`封装，失败时降级到内存存储    |
| **跨域不共享**   | 同源策略限制              | 使用PostMessage或后端同步                    |

---

===== 6、总结

```javascript
// 核心要点
window.localStorage   // BOM对象 - 持久存储，跨标签共享
window.sessionStorage // BOM对象 - 会话存储，标签隔离

// 都属于 window（BOM），不属于 document（DOM）
window.localStorage !== document.localStorage; // true
```

**选型建议**：
- **LocalStorage**：用户偏好设置、购物车、草稿箱（长期）
- **SessionStorage**：表单草稿、临时搜索条件、多步骤向导（短期）
- **IndexedDB**：大量结构化数据、离线应用（替代方案）

===== 二 JS 动画原理

**核心**：利用定时器，极短时间内修改元素的位置，利用人眼的视觉暂留形成动画。

**步骤**：

1.  获取元素当前位置。
2.  修改位置（+1 或 -1）。
3.  赋值回去。
4.  使用定时器重复执行。

**演示：匀速动画**

```javascript
            clearInterval(timer); // 到达终点停止
        } else {
            box.style.left = left + 5 + 'px'; // 每次走5px
        }
    }, 30); // 30毫秒刷新一次，约每秒33帧
}
```

=== S1：核心精讲（4课时）


==== 一：网络通信基础


===== 1. 客户端与服务器
*   **生活类比**：
    *   **客户端（浏览器）**：顾客（点菜的人）。
    *   **服务器**：后厨（做菜的人）。
    *   **HTTP协议**：服务员和后厨沟通的“菜单格式”，页面向后端数据库发送接收数据。
*   **请求与响应**：
    *   **Request（请求）**：浏览器发给服务器的内容（我要什么）。
    *   **Response（响应）**：服务器回给浏览器的内容（给你什么）。

===== 2. HTTP 状态码
不需要死记硬背，但要知道常见的：
*   `200`：成功。
*   `404`：找不到资源（路径错了）。
*   `500`：服务器内部错误（后厨着火了）。

---

==== 二：同步与异步


===== 1. 同步
*   代码排队执行，上一行没做完，下一行等着。
*   **缺点**：如果有一行代码要执行很久（比如下载大文件），页面会卡死。

===== 2. 异步
*   代码不排队，先执行后面的，等事情做完了再通知你。
*   **典型场景**：定时器、事件、网络请求。

**代码演示**：
```javascript
console.log("1. 开始点餐");

// 异步操作：模拟网络请求
setTimeout(function() {
    console.log("3. 饭菜做好了（异步结束）");
}, 2000);

console.log("2. 先去座位等着（代码继续执行）");

// 输出顺序：1 -> 2 -> 3
// 解释：2不会等3执行完才跑，这就是异步。
```

---

==== 三：Promise —— 异步操作的“承诺书”


===== 1. 为什么要用 Promise？（概念与背景）

**生活案例类比：**
*   **古代（回调函数时代）**：你早上去早餐店买包子。你付了钱，老板说“做好了叫你”。但如果有三个环节（做包子->装袋->拿筷子），你就得一直在那等着，老板喊一步你动一步，代码就会变成层层嵌套的“回调地狱”，像倒三角一样，极难维护。

*   **现代**：你点了外卖。下单后，APP给你一个“订单号”。你可以先去打游戏、看视频（异步操作）。等外卖到了（状态变为成功），APP弹窗提醒你去拿；如果送丢了（状态变为失败），APP退款。这个“订单号”就是 **Promise 对象**。

以前我们嵌套异步操作，代码像倒三角，极难维护。
```javascript
// 回调地狱示例（不要学）
setTimeout(() => {
    console.log("第一步");
    setTimeout(() => {
        console.log("第二步");
        setTimeout(() => {
            console.log("第三步");
        }, 1000);
    }, 1000);
}, 1000);
```

**核心概念**：
Promise 是一个对象，它代表了一个未来才会知道结果的事件。它有且只有三种状态：
1.  **Pending（进行中）**：比如外卖正在送。
2.  **Fulfilled / Resolved（已成功）**：外卖送到了。
3.  **Rejected（已失败）**：外卖送丢了。
*   **重要规则**：状态一旦改变（从 Pending 变为 Resolved 或 Rejected），就永久定格，不会再变了。

---

===== 2. Promise 基本语法与参数详解

**语法**：`new Promise(executor)`

*   **参数 `executor`**：这是一个执行器函数，它在 Promise 创建时立即执行。它接收两个参数：
    *   `resolve`：一个函数，调用它表示“成功”，将状态改为 Resolved。
    *   `reject`：一个函数，调用它表示“失败”，将状态改为 Rejected。

**代码演示：封装一个模拟读取文件的 Promise**

```javascript
// 1. 创建 Promise 对象
// resolve 和 reject 也是函数，由JS引擎内部提供
const p = new Promise((resolve, reject) => {
    console.log("1. 承诺开始执行..."); // 这行代码是同步执行的！

    // 模拟一个耗时操作（比如网络请求）
    setTimeout(() => {
        let success = true; // 模拟成功或失败

        if (success) {
            // 成功了，调用 resolve，并传递结果数据
            resolve("数据：这是一张美女图片");
        } else {
            // 失败了，调用 reject，并传递错误信息
            reject("错误：网络连接超时");
        }
    }, 1000);
});

console.log("2. 代码继续向下执行...");
```

---

===== 3. Promise 原型方法：处理结果

Promise 对象上有三个核心方法来处理结果。

**A. `.then(onFulfilled, onRejected)`**
*   **作用**：指定成功和失败的回调。
*   **参数**：
    *   第一个参数：成功的回调函数，接收 `resolve` 传来的数据。
    *   第二个参数：失败的回调函数（可选），接收 `reject` 传来的错误。

**B. `.catch(onRejected)`**
*   **作用**：专门用来捕获错误（推荐用法）。相当于 `.then(null, onRejected)`。

**C. `.finally(onSettled)`**
*   **作用**：无论成功还是失败，最终都会执行（比如隐藏加载动画）。

**代码演示：链式调用**

```javascript
p.then(result => {
    // 成功时的处理
    console.log("成功回调：", result); // 输出：数据：这是一张美女图片
}).catch(error => {
    // 失败时的处理
    console.log("失败回调：", error);
}).finally(() => {
    // 不管成功失败，都执行
    console.log("承诺结束，加载动画关闭");
});
```

---

===== 4. Promise 静态方法：多任务协作

**A. `Promise.all([])` —— 团队赛**
*   **规则**：所有 Promise 都成功，才返回成功；只要有一个失败，就返回失败。
*   **场景**：比如游戏资源加载，图片、音频、模型必须全部下载完才能开始。

**代码演示：**
```javascript
let p1 = new Promise((resolve) => setTimeout(() => resolve("图片加载完"), 1000));
let p2 = new Promise((resolve) => setTimeout(() => resolve("音频加载完"), 2000));
let p3 = new Promise((resolve) => setTimeout(() => resolve("模型加载完"), 1500));

// 等待所有完成
Promise.all([p1, p2, p3]).then(results => {
    console.log("全部加载成功：", results); 
    // 输出：['图片加载完', '音频加载完', '模型加载完'] (顺序固定)
}).catch(error => {
    console.log("有资源加载失败：", error);
});
```

**B. `Promise.race([])` —— 个人赛**
*   **规则**：谁先结束（无论成功失败），就返回谁的结果。
*   **场景**：请求超时机制。比如请求A和定时器B赛跑，如果B先跑完（时间到），就认为A超时。

**代码演示：**
```javascript
let request = new Promise(resolve => setTimeout(() => resolve("请求成功"), 2000));
let timeout = new Promise((_, reject) => setTimeout(() => reject("请求超时"), 1000));

Promise.race([request, timeout]).then(res => {
    console.log(res);
}).catch(err => {
    console.log(err); // 1秒后输出：请求超时
});
```

==== 四：Fetch API —— 现代网络请求标准


===== 1. Fetch 是什么？
*   Fetch 是浏览器原生提供的网络请求接口，用来替代老旧的 `XMLHttpRequest` (XHR)。
*   它基于 Promise 设计，不需要回调函数，写法非常优雅。

===== 2. Fetch 基本语法与参数详解

**语法**：`fetch(url, options).then(...).catch(...)`

**参数 1：`url` (必须)**
*   请求的地址字符串。

**参数 2：`options` (可选，配置对象)**
如果不写，默认是 GET 请求。常用配置项如下：
*   `method`：请求方法，如 `'GET'`、`'POST'`、`'DELETE'`。
*   `headers`：请求头对象。POST 请求通常需要设置 `'Content-Type': 'application/json'`。
*   `body`：请求体。发送数据时使用（GET 请求没有 body）。通常是 JSON 字符串。

---

===== 3. 示例一：GET 请求（获取数据）

**场景**：请求一个公开的笑话接口。

**代码演示**：
```javascript
// 1. 直接调用 fetch，返回一个 Promise
fetch('https://api.apiopen.top/getJoke?page=1&count=2&type=text')
  .then(response => {
      // 注意：这里的 response 是一个 Response 对象
      // 它包含了状态码、头信息等，但不包含真正的数据

      console.log(response.status); // 200 (HTTP状态码)

      // 2. 我们需要调用 .json() 方法来解析数据体
      // .json() 方法也是一个异步操作，返回一个新的 Promise
      return response.json();
  })
  .then(data => {
      // 3. 这里拿到的才是真正的 JSON 数据
      console.log("最终数据：", data);
      // 可以在这里进行页面渲染
  })
  .catch(error => {
      // 只有网络断开或域名错误才会进这里
      console.log("网络错误：", error);
  });
```

**重点解释**：
为什么要写两个 `.then`？
*   第一个 `.then` 拿到的是 HTTP 响应报文（包装盒）。
*   `response.json()` 是拆开包装盒（异步过程）。
*   第二个 `.then` 拿到的是盒子里面的商品。

---

===== 4. 示例二：POST 请求（提交数据）

**场景**：模拟用户登录，提交账号密码。

**代码演示**：
```javascript
// 准备数据
let userData = {
    username: "admin",
    password: "123456"
};

fetch('https://mock-api.com/login', {
    // 1. 设置方法
    method: 'POST',

    // 2. 设置请求头
    headers: {
        'Content-Type': 'application/json' // 告诉服务器，我发的是JSON格式数据
    },

    // 3. 设置请求体
    // body 只能接收字符串，所以要用 JSON.stringify() 把对象转成字符串
    body: JSON.stringify(userData)
})
.then(response => response.json())
.then(result => {
    if (result.code === 200) {
        console.log("登录成功");
    } else {
        console.log("登录失败：", result.message);
    }
})
.catch(error => console.error("Error:", error));
```

---

===== 5. 示例三：Fetch 的错误处理（陷阱！）

**重要概念**：
Fetch 与 jQuery 不同。在 Fetch 中，**HTTP 状态码 404 或 500 并不会触发 `.catch`**。只要服务器有响应，Promise 就是 Resolved 状态。`.catch` 只捕获网络故障（如断网、DNS解析失败）。

**正确处理方式**：手动检查 `response.ok`。

**代码演示**：
```javascript
fetch('https://api.com/not-found-page') // 一个不存在的地址
  .then(response => {
      // response.ok 是一个布尔值，状态码 200-299 之间为 true
      if (response.ok) {
          return response.json();
      } else {
          // 手动抛出错误，跳转到 catch
          throw new Error(`服务器错误，状态码：${response.status}`);
      }
  })
  .then(data => console.log(data))
  .catch(error => {
      // 这里可以捕获到 404 错误了
      console.log(error.message); // 输出：服务器错误，状态码：404
  });
```

---

===== 6. 示例四：结合 Async/Await 的终极写法

这才是现代企业开发的标准写法，代码像同步一样清晰。

**代码演示**：
```javascript
async function getData() {
    try {
        // 1. 发送请求，await 等待响应对象
        let response = await fetch('https://api.apiopen.top/getJoke');

        // 2. 手动检查状态
        if (!response.ok) {
            throw new Error("请求失败");
        }

        // 3. 解析数据
        let data = await response.json();

        // 4. 使用数据
        console.log("获取到的笑话：", data.result);

    } catch (error) {
        // 统一处理错误
        console.log("发生错误：", error);
    }
}

getData();
```

---

**教学总结**：
1.  **Promise** 是解决异步顺序执行和回调地狱的钥匙，必须掌握 `.then`、`.catch` 和 `all` 的用法。
2.  **Fetch** 是获取网络数据的工具，必须理解它默认不报错 404 的特性，以及 `.json()` 解析数据的步骤。
3.  **Async/Await** 是 Fetch 的最佳伴侣，让我们用写同步代码的方式写异步逻辑。

通过这部分的详细讲解和丰富示例，学生应该能清晰地理解网络请求的完整生命周期，并具备处理各种异常情况的能力。

**场景**：我们的项目数据一直写死在代码里。今天，我们要模拟真实的“前后端分离”。

*   **语法**：在函数前加 `async` 关键字。
*   **返回值**：返回的一定是一个 Promise 对象。

**代码演示**：
```javascript
async function hello() {
    return "你好"; 
    // 虽然返回的是字符串，但实际上函数返回的是 Promise.resolve("你好")
}

});
```

---

==== 二：Await 表达式


===== 1. await 是什么？
`await` 只能用在 `async` 函数内部。
它会**暂停**函数的执行，直到 Promise 返回结果。这就是“以同步的方式写异步”。

**代码演示**：
```javascript
function getTime() {
    return new Promise(resolve => {
        setTimeout(() => {
            resolve("时间到了");
        }, 2000);
    });
}

async function test() {
    console.log("开始");

    // 这一行会停住，直到 getTime 返回结果
    let result = await getTime(); 
    console.log(result); // 2秒后打印：时间到了

    console.log("结束");
}

test();
```
**对比**：这比写 `.then()` 看起来清晰太多了！

==== 三：错误处理


===== 1. try...catch
如果 `await` 后面的 Promise 失败了，会抛出异常，我们需要用 `try...catch` 捕获。

```javascript
async function loadData() {
    try {
        let res = await fetch('./data.json');
        let data = await res.json();
        console.log("数据：", data);
    } catch (error) {
        console.log("捕获到错误：", error);
    }
}

loadData();
```

*   `private`：私有，只能在类里面用，外面不能访问。

```typescript
class Person {

        return this.age; // 内部可以访问
    }
}

let p = new Person("张三", 18);
console.log(p.name); // OK
// console.log(p.age); // 报错！私有属性外部不可访问
console.log(p.getAge()); // OK
```

**涵盖知识点**：

===== 1. 需求描述
鼠标移入星星时，当前星星及之前的星星变黄；鼠标移出恢复默认状态；点击时锁定评分。

```html
    <!-- 5个星星，使用 unicode 字符 -->
    <span class="star text-4xl text-gray-300 cursor-pointer transition-colors duration-200 hover:text-yellow-400">★</span>
    <span class="star text-4xl text-gray-300 cursor-pointer transition-colors duration-200 hover:text-yellow-400">★</span>
    <span class="star text-4xl text-gray-300 cursor-pointer transition-colors duration-200 hover:text-yellow-400">★</span>
    <span class="star text-4xl text-gray-300 cursor-pointer transition-colors duration-200 hover:text-yellow-400">★</span>
    <span class="star text-4xl text-gray-300 cursor-pointer transition-colors duration-200 hover:text-yellow-400">★</span>
```

===== 3. jQuery 逻辑
```javascript
$(function() {
    let currentScore = 0; // 记录点击后的分数

    // 1. 鼠标移入效果
    $('#rating').on('mouseenter', '.star', function() {
        // 获取当前星星的索引 (0-4)
        let index = $(this).index();

        // 将当前及之前的星星变黄
        $(this).prevAll().addBack().removeClass('text-gray-300').addClass('text-yellow-400');

        // 将之后的星星变灰 (为了处理鼠标回退的情况)
        $(this).nextAll().removeClass('text-yellow-400').addClass('text-gray-300');
    });

    // 2. 鼠标移出整个容器效果
    $('#rating').on('mouseleave', function() {
        // 如果有点击过，恢复到点击的状态；否则全灰
        if (currentScore === 0) {
            $('.star').removeClass('text-yellow-400').addClass('text-gray-300');
        } else {
            // 根据保存的分数重新点亮
            $('.star').each(function(i) {
                if (i < currentScore) {
                    $(this).removeClass('text-gray-300').addClass('text-yellow-400');
                } else {
                    $(this).removeClass('text-yellow-400').addClass('text-gray-300');
                }
            });
        }
    });

    // 3. 点击锁定分数
    $('#rating').on('click', '.star', function() {
        currentScore = $(this).index() + 1; // 分数 = 索引 + 1
        $('#ratingText').text(`您打分：${currentScore} 分`);
    });
});
```

```javascript
$(function() {
    // 封装计算总价的函数
    function calculateTotal() {
        let total = 0;
        // 遍历每一个商品行
        $('.item').each(function() {
            // 获取单价和数量
            // 注意：.text() 获取的是文本，需要转数字
            let price = parseFloat($(this).find('.price span').text());
            let num = parseInt($(this).find('.quantity').val());

            // 计算小计
            let subTotal = price * num;

            // 设置小计 (保留两位小数)
            $(this).find('.sub-total span').text(subTotal.toFixed(2));

            // 累加总价
            total += subTotal;
        });

        // 设置总价
        $('#totalPrice').text('￥' + total.toFixed(2));
    }

    // 1. 加号点击事件
    $('#cartList').on('click', '.plus', function() {
        // 获取当前输入框对象
        let $input = $(this).siblings('.quantity');
        let num = parseInt($input.val());
        $input.val(num + 1);

        // 重新计算
        calculateTotal();
    });

    // 2. 减号点击事件
    $('#cartList').on('click', '.minus', function() {
        let $input = $(this).siblings('.quantity');
        let num = parseInt($input.val());

        if (num > 1) {
            $input.val(num - 1);
            calculateTotal();
        } else {
            alert("数量不能少于1");
        }
    });

    // 初始化计算一次
    calculateTotal();
});
```

===== 说明

**1. 关于 jQuery 选择器的技巧**：
*   在购物车案例中，使用了 `$(this).siblings('.quantity')`，这是非常典型的“相对查找”思维。不要在循环里给每个 input 加 ID，而是通过按钮的位置去找旁边的元素，这在动态生成的列表中尤为重要。

**2. 关于 Tailwind 的响应式**：
*   在手风琴案例中，你可以尝试在小屏幕下改变布局。例如给容器加上 `md:w-1/2 w-full`，这能让学生直观感受响应式开发的便捷。

**3. 关于数据类型**：
*   购物车案例中必须强调 `parseFloat` 和 `parseInt`。JS 从 HTML 取出来的永远是字符串，直接相加会变成字符串拼接（如 "59" + 1 = "591"），这是初学者最容易犯的错。

```html
                </div>
            </form>
        </div>
    </div>

    <script>
        // ==========================================
        // 1. 数据初始化
        // ==========================================
        let goodsData = [
            { id: 1, name: "高等数学", price: 15, category: "书籍" },
            { id: 2, name: "二手吉他", price: 200, category: "乐器" },
            { id: 3, name: "台灯", price: 35, category: "生活" }
        ];

        let sortState = { key: null, order: 'asc' };
        const $formModal = $('#formModal');
        const $form = $('#dataForm');
        const $confirmModal = $('#confirmModal'); // 确认框

        // ==========================================
        // 2. 核心渲染函数
        // ==========================================
        function renderTable(data) {
            let list = data || goodsData;
            $('#goodsTable').empty();

            if (list.length === 0) {
                $('#goodsTable').append('<tr><td colspan="5" class="p-4 text-center text-gray-400">暂无数据</td></tr>');
                return;
            }

            $.each(list, function(i, item) {
                let tr = `
                    <tr class="hover:bg-gray-50">
                        <td class="p-3 border">${item.id}</td>
                        <td class="p-3 border">${item.name}</td>
                        <td class="p-3 border text-red-500 font-bold">￥${item.price.toFixed(2)}</td>
                        <td class="p-3 border"><span class="bg-blue-100 text-blue-700 text-xs px-2 py-1 rounded">${item.category}</span></td>
                        <td class="p-3 border">
                            <button class="edit-btn bg-yellow-400 px-2 py-1 rounded text-white text-sm hover:bg-yellow-500" data-id="${item.id}">编辑</button>
                            <button class="del-btn bg-red-400 px-2 py-1 rounded text-white text-sm hover:bg-red-500 ml-1" data-id="${item.id}">删除</button>
                        </td>
                    </tr>
                `;
                $('#goodsTable').append(tr);
            });
        }

        renderTable();

        // ==========================================
        // 3. 自定义确认框逻辑 (替代原生 confirm)
        // ==========================================
        function showConfirm(message, callback) {
            $('#confirmMsg').text(message);
            $confirmModal.removeClass('hidden');

            // 解绑旧事件，防止重复绑定
            $('#confirmOk').off('click').on('click', function() {
                $confirmModal.addClass('hidden');
                if (typeof callback === 'function') {
                    callback(); // 执行回调
                }
            });

            $('#confirmCancel').off('click').on('click', function() {
                $confirmModal.addClass('hidden');
            });
        }

        // ==========================================
        // 4. 删除功能 (使用自定义确认框)
        // ==========================================
        $('#goodsTable').on('click', '.del-btn', function() {
            let id = $(this).data('id');

            // 调用自定义确认框
            showConfirm('确定要删除该商品吗？', function() {
                // 用户点击确定后执行
                goodsData = goodsData.filter(item => item.id !== id);
                $('#search').val('');
                renderTable();
            });
        });

        // ==========================================
        // 5. 搜索功能
        // ==========================================
        $('#search').on('input', function() {
            let keyword = $(this).val().trim().toLowerCase();
            if (!keyword) { renderTable(); return; }
            let filtered = goodsData.filter(item => 
                item.name.toLowerCase().includes(keyword) || 
                item.category.toLowerCase().includes(keyword)
            );
            renderTable(filtered);
        });

        // ==========================================
        // 6. 排序功能
        // ==========================================
        $('th[data-key]').click(function() {
            let key = $(this).data('key');
            if (sortState.key === key) {
                sortState.order = (sortState.order === 'asc') ? 'desc' : 'asc';
            } else {
                sortState.key = key;
                sortState.order = 'asc';
            }
            $('th[data-key] .sort-icon').text('↕').removeClass('active-sort');
            $(this).find('.sort-icon').text(sortState.order === 'asc' ? '↑' : '↓').addClass('active-sort');

            goodsData.sort((a, b) => {
                let valA = a[key], valB = b[key];
                if (typeof valA === 'number') return sortState.order === 'asc' ? valA - valB : valB - valA;
                if (typeof valA === 'string') return sortState.order === 'asc' ? valA.localeCompare(valB) : valB.localeCompare(valA);
                return 0;
            });
            $('#search').val('');
            renderTable();
        });

        // ==========================================
        // 7. 弹窗控制 (新增 & 编辑)
        // ==========================================
        function openFormModal(isEdit, item = {}) {
            if (isEdit) {
                $('#modalTitle').text('编辑商品');
                $('#editId').val(item.id);
                $('#inputName').val(item.name);
                $('#inputPrice').val(item.price);
                $('#inputCategory').val(item.category);
            } else {
                $('#modalTitle').text('新增商品');
                $('#editId').val('');
                $form[0].reset();
            }
            $formModal.removeClass('hidden');
        }

        function closeFormModal() {
            $formModal.addClass('hidden');
            $form[0].reset();
            $('#editId').val('');
        }

        $('#addBtn').click(function() {
            openFormModal(false);
        });

        $('#goodsTable').on('click', '.edit-btn', function() {
            let id = $(this).data('id');
            let item = goodsData.find(g => g.id === id);
            if (item) {
                openFormModal(true, item);
            }
        });

        $('#cancelBtn').click(closeFormModal);

        // ==========================================
        // 8. 表单提交
        // ==========================================
        $form.submit(function(e) {
            e.preventDefault();

            let id = $('#editId').val();
            let name = $('#inputName').val().trim();
            let price = parseFloat($('#inputPrice').val());
            let category = $('#inputCategory').val();

            if (!name || isNaN(price)) {
                // 这里也可以用自定义弹窗，为了简单暂用 alert，如果报错同理需替换
                // 暂时简单处理：直接修改文本颜色提示
                if(!name) $('#inputName').css('border-color', 'red'); 
                return;
            }

            if (id) {
                let index = goodsData.findIndex(item => item.id == id);
                if (index !== -1) {
                    goodsData[index] = { 
                        id: Number(id), 
                        name: name, 
                        price: price, 
                        category: category 
                    };
                }
            } else {
                let maxId = goodsData.length > 0 ? Math.max(...goodsData.map(item => item.id)) : 0;
                goodsData.push({
                    id: maxId + 1,
                    name: name,
                    price: price,
                    category: category
                });
            }

            closeFormModal();
            $('#search').val('');
            renderTable();
        });

    </script>
</body>
</html>
```
